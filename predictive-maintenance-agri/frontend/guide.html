Manuel de l'Opérateur

Guide complet des paramètres, unités techniques et interprétation des résultats de l'Intelligence Artificielle.

1. Comment utiliser l'interface

Le dashboard centralise les données provenant de vos pompes d'irrigation connectées (IoT) et utilise un modèle de Machine Learning (Random Forest) pour prédire les défaillances.

Saisie des données

Les champs acceptent des valeurs numériques précises. Vous pouvez utiliser les valeurs par défaut pour tester le comportement du modèle.

Diagnostic Instantané

L'analyse prend moins de 500ms. Le modèle traite 14 variables (8 capteurs + contextes temporels + statistiques mobiles) pour générer le score.

2. Paramètres et Unités

Chaque capteur doit être calibré selon les unités suivantes pour garantir la précision du diagnostic.

Capteur / Paramètre Unité Description technique
Température Moteur °C Température interne du bobinage. Une valeur > 75°C indique souvent une surcharge.
Vibration mm/s Vitesse vibratoire globale (RMS). Crucial pour détecter l'usure des roulements ou le désalignement.
Courant Électrique A (Ampères) Intensité consommée. Une hausse soudaine peut indiquer un blocage mécanique (pompe obstruée).
Voltage V (Volts) Tension d'alimentation. Doit rester stable (ex: 220V ou 380V) pour éviter d'endommager l'électronique.
Pression Eau bar Pression en sortie de pompe. Une chute de pression indique une fuite ou une cavitation.
Débit Eau L/min Volume d'eau pompé par minute. Permet de vérifier l'efficacité hydraulique.
Vitesse (RPM) tr/min Rotations par minute du moteur. Un RPM instable signale souvent un problème de variateur.
Heures Fonct. h Compteur horaire total. Utilisé par l'IA pour pondérer l'usure naturelle de l'équipement.

3. Interprétation des Sorties

Le résultat principal est un Pourcentage de Risque. Voici comment le lire :

Niveau Risque Signification & Action
OPTIMISÉ 0 - 20% État parfait. Les capteurs sont dans les zones de haute efficacité.
NORMALE 20 - 50% Fonctionnement nominal. Aucune action immédiate, surveillance standard.
NEED FIX 50 - 80% Alerte Préventive. Signes de fatigue détectés. Planifiez une inspection sous 7 jours.
CRITIQUE 80 - 100% Danger Imminent. Risque de casse majeur. Arrêt de l'équipement et maintenance immédiate.

Signaux Spéciaux

  • Notification : Un message généré dynamiquement qui explique la décision du modèle.
  • Recommandation : L'action prioritaire à effectuer par l'équipe de maintenance.
  • Hors Limites (Warnings) : Indique que les données saisies sont en dehors des plages connues par l'IA (Data Drift). Le diagnostic peut être moins fiable.

4. Architecture de Sécurité (MLOps)

Notre système n'est pas une "boîte noire". Il combine l'intelligence artificielle avec des règles métiers strictes :

Validation Statistique

Chaque entrée est comparée aux statistiques de référence (Min/Max/Moyenne). Si une valeur est absurde, le système vous alerte.

Override de Sécurité

Si la vibration dépasse 2x le maximum autorisé ou si la température excède +20% de la limite, le système force l'état CRITIQUE par sécurité, même si l'IA ne le détecte pas.

IMPORTANT

Si vous voyez le badge SÉCURITÉ : CRITIQUE, ignorez les autres indicateurs et procédez à la mise hors tension de l'équipement. C'est une protection matérielle active.
